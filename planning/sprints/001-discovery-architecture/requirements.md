# Sprint 001 Requirements: Discovery and Architecture

## Goal

Create the operating system for SOKA Baru before implementation begins.

## Business Objective

Turn mixed inputs from SOKA Lama, the Super Apps Sekolah deck, and the 120x Operators Kit into a clear, documented, sprint-based plan.

## In Scope

- Establish the project operating folder.
- Capture source materials.
- Record durable decisions.
- Define grill sessions.
- Clarify domain terminology.
- Draft initial risks and open questions.

## Out of Scope

- Writing production application code.
- Migrating SOKA Lama code.
- Finalizing UI design.
- Choosing every module in the roadmap.

## Acceptance Summary

This sprint is complete when the project can be restarted from the folder without relying on chat memory.
