# Module Classification

Session 4 classifies modules from the Super Apps Sekolah deck, SOKA Lama, and new SOKA Baru scope.

## Labels

| Label | Meaning |
|---|---|
| Now | Belongs in MVP. |
| Later | Valid product direction, but not MVP. |
| Never | Outside SOKA's product boundary unless strategy changes. |
| Rebuild Fresh | Useful concept, but do not copy the old implementation. |

## Classified Modules

| Module | Classification | Reason |
|---|---|---|
| Absensi | Now | First adoption wedge, daily anchor, parent transparency signal, and objective input for student status. |
| Pesan Ortu | Now | Part of the daily loop and the main responsiveness channel between teacher and parents. |
| Pengumuman kelas/sekolah | Later | Useful, but can expand into broadcast segmentation, templates, approvals, and school-wide communication workflows. |
| Nilai dasar / nilai terhadap KKM | Now | Supports parent transparency and objective student status without requiring full raport workflow. |
| Raport lengkap / finalisasi semester | Later | Valid product direction, but brings semester formats, descriptions, validation, printing, approvals, and curriculum-specific complexity. |
| Notifikasi transaksional inti | Now | Needed for parent value: non-present/late attendance, new messages, published student notes, and new grades. |
| Notifikasi broadcast/campaign/reminder kompleks | Later | Can expand into preferences, scheduled sends, campaigns, templates, and analytics. |
| Catatan Siswa kualitatif + publish status | Now | Supports teacher documentation and parent communication without behavior scoring. |
| Manajemen kasus / intervensi / SLA BK | Never for MVP | Too heavy for wali kelas SD MVP; only reconsider later if targeting schools with active counseling/BK workflows. |
| Pembayaran / SPP online | Later | Potentially valuable, but needs payment gateway, reconciliation, refunds, admin fees, finance ownership, receipts, and dispute handling. |
| Admin onboarding minimal | Now | Required to make MVP usable: create school, import students, link parents and children, assign classes and teachers. |
| TU module lengkap | Later | Valid future direction, but data induk besar, documents, letters, institutional calendar, finance, and staff attendance would pull SOKA toward enterprise SIS scope. |
| Sales/report ringkas untuk pimpinan | Later | Useful for buyers and renewal, but not part of the first daily operational MVP. |
| Dashboard analytics kepsek penuh | Later | Valid future direction, but would pull MVP toward BI, cross-class analytics, and leadership workflows. |
| KPI guru / evaluasi performa guru | Never for MVP | Too sensitive and shifts SOKA toward staff evaluation before the trust loop is mature. |
| Tugas Digital | Rebuild Fresh / Later | Valid Phase 2 concept with role Siswa, but old implementation should not be copied. Rebuild around teacher creates, student submits, teacher grades, parent sees summary. |
| Repositori Materi | Rebuild Fresh / Later | Valid Phase 2 concept with role Siswa, but should be rebuilt around real access, ownership, and student consumption instead of old mock flows. |
| Ekskul | Later | Potentially useful for broader school life, but not part of the first school-parent trust loop. |
| Bank Soal | Later or Never for MVP | Assessment-heavy and closer to LMS scope than the first operating-layer MVP. |
| Forum Diskusi | Never for MVP | Not core for SD trust loop and introduces moderation/safety complexity. |
| Laporan Instansi | Never for MVP | Pulls SOKA toward government reporting and enterprise SIS scope. |
